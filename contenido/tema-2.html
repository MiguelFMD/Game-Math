Tema 2: Recursividad en videojuegos y programación

1. ¿Qué se entiende por recursividad?

La recursividad (o recursión) es aquella propiedad que posee una función por la cual dicha función puede llamarse a sí misma. Se puede utilizar como una alternativa a la iteración (los bucles clásicos como for o while).

Una solución recursiva es normalmente menos eficiente en términos de tiempo de computadora que una solución iterativa, debido a las operaciones auxiliares de memoria que conllevan las llamadas suplementarias. Sin embargo, en muchas circunstancias permite especificar soluciones naturales y sencillas a problemas que serían muy difíciles de resolver de otra forma.

2. Estructura de la recursividad

En matemáticas existen numerosas funciones que tienen carácter recursivo; de igual modo numerosas circunstancias y situaciones de la vida ordinaria tienen carácter recursivo, por ejemplo, el factorial de un número o Fibonacci.

Hasta el momento, casi siempre hemos visto métodos que llaman a otras funciones distintas. Por ejemplo:

private void Func1() {
    print("Func1 llamada");
    Func2();
}

private void Func2() {
    print("Func2 llamada");
}

La recursividad ocurre cuando una función se llama a sí misma para resolver un problema más pequeño del mismo tipo. Para que no se ejecute infinitamente, requiere dos partes fundamentales:

  • Caso base: El momento en que se detiene la recursión (la condición de salida).
  • Caso recursivo: La llamada a sí misma con un problema reducido.
La metáfora de la recursividad

"Es como mirarse en dos espejos enfrentados: la imagen se repite una y otra vez, hasta que algo (la distancia o el límite visual) la detiene."

private void FuncRecursiva() {
    // 1. Si llegamos al caso base, acabamos
    return;
    
    // 2. Si no, se continua con la recursión:
    FuncRecursiva();
}

3. Ejemplos clásicos: El Factorial

El factorial de un entero no negativo $n$, escrito $n!$, es el producto de todos los números hasta 1:

$$n! = n \cdot (n-1) \cdot (n-2) \dots 1$$

en el cual:

$$0! = 1$$ $$1! = 1$$ $$2! = 2*1 = 2*1!$$ $$3! = 3*2*1 = 3*2!$$ $$4! = 4*3*2*1 = 4*3! ...$$

así queda:

$$5! = 5 * 4 * 3 * 2 * 1 = 5 * 4! = 120$$

Sabiendo que, por definición, $0! = 1$ y $1! = 1$, podemos deducir una definición recursiva:

$$n! = n \cdot (n-1)! \quad \text{para } n > 1$$

Solución Iterativa

El algoritmo que resuelve el factorial de forma iterativa utiliza un bucle. Como vimos en el Tema 1, este algoritmo tiene un coste temporal $O(n)$, ya que es directamente proporcional al tamaño del número $n$.

int factorial_iterativo(int n) {
    int resultado = 1;
    for (int i = 2; i <= n; i++) {
        resultado *= i;
    }
    return resultado;
}

Solución Recursiva

El método de definición de una función en términos de sí misma se llama en matemáticas definición inductiva. El caso base es esencial para detener la cadena de llamadas.

int factorial_recursivo(int n) {
    // Condición de terminación (Caso Base)
    if (n == 1 || n == 0) return 1;
    
    // Llamada recursiva
    return n * factorial_recursivo(n - 1);
}
Condición de terminación y Desbordamiento

Si no se establece una condición de terminación, el subprograma continuaría indefinidamente llamándose a sí mismo hasta agotar la memoria de la Pila de Llamadas (Stack Overflow). Además, dado que el valor de un factorial aumenta de forma vertiginosa, es conveniente en el diseño definir el tipo de dato como un valor real (ej. long o double) para evitar desbordamientos de la variable.

4. Más ejemplos: Producto y Fibonacci

Producto de números naturales

El producto $a \cdot b$, donde $a$ y $b$ son enteros positivos, se puede definir recursivamente como la suma repetida de $a$.

  • Caso base: $a \cdot b = a \quad \text{si } b = 1$
  • Caso recursivo: $a \cdot b = a + (a \cdot (b-1)) \quad \text{si } b > 1$
int producto(int a, int b) {
    if (b == 1) return a;
    return a + producto(a, b - 1);
}

Serie de Fibonacci

La serie comienza con 0 y 1, y cada elemento posterior es la suma de los dos anteriores. La definición recursiva requiere dos llamadas a sí misma por cada iteración:

int fibonacci_recursivo(int n) {
    if (n <= 1) return n;
    return fibonacci_recursivo(n - 1) + fibonacci_recursivo(n - 2);
}

Nota: Como vimos en el análisis de costes, esta doble llamada genera un árbol exponencial, dándole una complejidad temporal altísima de $O(2^n)$.

5. Ejercicios de recursividad

Implementa las siguientes funciones utilizando metodologías recursivas en C#:

  1. Contar las cifras de un entero no negativo $n$. (Pista: si divides por división entera entre 10, le vas quitando cifras al número. Ej: 1234 / 10 = 123).
  2. Contar las veces que una cifra $c$ aparece en un entero no negativo $n$. (Pista: piensa cuál es el resultado de 1234 % 10).
  3. Averiguar si todas las cifras de un entero no negativo $n$ son pares.
  4. Averiguar si un dato $d$ aparece en un vector no ordenado $v$.
  5. Contar las veces que un dato $d$ aparece en un vector no ordenado $v$.
  6. Mostrar en la consola los elementos de un vector no ordenado $v$ en orden natural (del primero al último), uno en cada línea.
  7. Mostrar en la consola los elementos de un vector no ordenado $v$ en orden inverso (del último al primero), uno en cada línea.
  8. Averiguar si un vector no vacío $v$ está ordenado de menor a mayor.
  9. Encontrar el menor elemento en un vector no ordenado y no vacío $v$.
  10. Averiguar si todos los elementos de un vector no ordenado y no vacío $v$ son mayores que un dato $d$.
